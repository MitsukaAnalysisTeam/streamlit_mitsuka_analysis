# 年間分析ページ 詳細設計書

## 1. 目的

日次・月次の変動だけでは把握しづらい中長期の経営傾向を、年次で比較できるようにする。  
対象期間を `2022年以降〜最新年` とし、最新年を含めた推移を同一画面で確認可能にする。

提供する指標は以下の5つ。

- 合計客数
- 合計売上
- 平均客数（日次平均）
- 平均売上（日次平均）
- 平均客単価（日次平均）

## 2. 設計方針

- 既存の日報分析基盤を再利用し、データ定義の一貫性を保つ
- 集計ロジックと描画ロジックを分離し、保守性を高める
- UIはボタン操作を排し、1ページで5グラフを固定表示する
- 表示年は固定値ではなく、データに応じて動的に確定する

## 3. データモデル

### 3.1 入力データ

年間分析の入力元は Google スプレッドシート `日報_all` である。  
年間分析は直接スプレッドシートを読むのではなく、日報分析基盤で整形済みの
`年 -> 月 -> DataFrame` 構造を受け取って集計する。

入力取得フロー:

1. サービスアカウントで Google API 認証する
2. Drive API で対象フォルダ内のスプレッドシート名 `日報_all` を検索する
3. 検索結果から Spreadsheet ID を取得する
4. Spreadsheet ID でスプレッドシートを開き、`シート1` を取得する
5. ワークシートを DataFrame 化し、`日付` 列を datetime 化してインデックス化する
6. `年 -> 月 -> DataFrame` に再構成して年間分析へ受け渡す

検索条件（論理値）:

- `folder_id = "1zPbemG8PafxBNUAmsIkqTNM_4RSBh3lZ"`
- `spreadsheet_name = "日報_all"`
- `worksheet_name = "シート1"`

年間分析で参照する必須列:

- `1日総客数`
- `1日総売上`
- `1日客単価`

前処理の前提:

- 空文字や空白のみの値は欠損として扱う
- 数値列は文字列混在の可能性があるため、年次集計時に数値変換する
- 月データが空の場合は当該月をスキップする

### 3.2 出力データ（年間サマリ）

インデックス: `year`（文字列）  
カラム:

- `合計客数`
- `合計売上`
- `平均客数`
- `平均売上`
- `平均客単価`

## 4. 集計仕様

### 4.1 年の対象範囲

- 下限: 2022年
- 上限: データ内の最新年（動的）
- 途中年（例: 2026年）も、存在する日次データのみで集計対象とする

### 4.2 算出式

各年の全日次レコードを連結した上で、以下を算出する。

- `合計客数 = sum(1日総客数)`
- `合計売上 = sum(1日総売上)`
- `平均客数 = mean(1日総客数)`
- `平均売上 = mean(1日総売上)`
- `平均客単価 = mean(1日客単価)`

補足:

- 平均は月平均ではなく日次平均
- 数値列は非数値混在に備えて数値変換を行う
- 空月や必要列不足の月はスキップする

## 5. 画面仕様

### 5.1 ページ構成

ページ名: 年間分析  
表示順（固定）:

1. 合計客数
2. 合計売上
3. 平均客数
4. 平均売上
5. 平均客単価

### 5.2 グラフ仕様

- 形式: 棒グラフ
- x軸: 年
- y軸: 指標に応じた単位（人/円）
- 色: 最新年のみ強調色、その他は同系色
- 表示: 1ページ内に縦積みで連続表示
- 凡例: 各グラフで指標名を凡例として表示
- 数値ラベル: 各棒グラフの上に太字で表示
- ホバー表示: 年と指標値を詳細表示

### 5.3 補足情報表示

- ページ上部に、最新年が何月まで集計対象かを表示する  
  例: `最新年データ範囲: 2026年4月まで（集計対象）`

## 6. コンポーネント責務

### 6.1 年次集計コンポーネント

責務:

- 入力の年/月辞書データから年間サマリDataFrameを生成する
- 対象年フィルタ、欠損対応、数値変換を一元管理する

公開インターフェース:

- `build_yearly_summary(df_dic: dict, start_year: int = 2022) -> pd.DataFrame`

### 6.2 年次チャートコンポーネント

責務:

- 年間サマリDataFrameを受け取り、単一指標の棒グラフを描画する
- 通貨/人数フォーマットと強調色ルールを統一する

公開インターフェース:

- `plot_yearly_bar(df_yearly, metric_col, y_label, currency=False, people=False)`

## 7. エラーハンドリング

- 年間サマリが空の場合は警告を表示し、描画処理を中断
- 指標カラムが不足する場合は対象グラフのみ警告表示
- 月データ不備は月単位でスキップし、全体停止を避ける

## 8. 非機能要件

- 既存分析と同じデータ定義で整合すること
- 同一データに対して再現性のある集計結果を返すこと
- 途中年データでも破綻しないこと

## 9. 受け入れ条件

- 2022年から最新年までが昇順で表示される
- 5グラフが選択UIなしで1ページに表示される
- 平均指標が日次平均として算出される
- 欠損・空月があってもページ表示が継続される

## 10. 将来拡張

- 前年比（YoY）表示
- 目標値ラインと達成率表示
- 属性別（平日/祝日など）年次比較
